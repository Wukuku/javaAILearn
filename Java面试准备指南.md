| 问题 | 如果是 | 准备时间 |

## 二、Java 核心考点（按重要程度）

### 必须掌握（高频）

- **JVM**：内存模型、GC 算法（G1/ZGC）、类加载机制、OOM 排查
- **并发**：`synchronized` vs `ReentrantLock`、线程池参数、`volatile`、CAS 原理
- **集合**：`HashMap` 扩容原理（负载因子/红黑树）、`ConcurrentHashMap` 分段锁
- **Spring**：Bean 生命周期、AOP 原理（动态代理）、事务传播机制
- **MySQL**：索引（B+树结构）、事务隔离级别、explain 分析慢查询、锁机制
- **Redis**：数据结构选型、缓存穿透/击穿/雪崩解决方案、分布式锁

### 加分项（结合项目经验更好说）

- **Spring AI 项目**：RAG 原理、Agent/ReAct 模式、流式输出 ← **差异化竞争力**
- **微服务**：Nacos 注册发现、Sentinel 限流、Gateway 路由基本原理

## 三、算法怎么准备

> 普通公司：基本不考，或只考简单题
> 大厂：必考，通常 2-3 轮算法面

### 刷题路线（从零开始约 6-8 周）

**第 1-2 周：基础数据结构**
- 数组、链表、栈、队列、哈希表
- 目标：LeetCode Easy 全部通关

**第 3-4 周：核心算法**
- 双指针、滑动窗口、二分查找、BFS/DFS
- 目标：LeetCode Medium 完成 60%

**第 5-6 周：进阶专题**
- 动态规划（背包 / 子序列 / 路径问题）
- 树的遍历 + 回溯算法

**第 7-8 周：模拟面试**
- 每天 2 题，限时 30 分钟作答
- LeetCode Hot 100 完整过一遍

### 刷题量参考

| 目标 | 刷题量 |
|---|---|
| 普通公司 | Hot 100 刷完即可 |
| 大厂 | 200-300 题起步 |

## 四、分场景备战计划

### 1 个月内要面试

1. 算法暂时放下，优先背熟 Java/Spring/MySQL 八股文
2. 把 Spring AI 项目讲清楚（RAG、Agent、LLM-as-Judge 都是亮点）
3. 算法只刷 Easy，保证基础题不翻车

### 有 2-3 个月充裕时间

| 阶段 | 重点 |
|---|---|
| 第 1 个月 | 算法刷题 + JVM/并发深挖 |
| 第 2 个月 | Spring + MySQL + Redis + 项目深挖 |
| 第 3 个月 | 模拟面试 + 查漏补缺 + 复盘 |

## 五、项目差异化亮点（结合 javaAILearn）

面试时可以主动提，大多数 Java 候选人没有 AI 工程经验：

- **RAG 落地**：文档分片、向量检索、topK/阈值调优
- **Agent 实现**：ReAct 模式、Function Calling 循环、CoT Prompt
- **LLM-as-Judge**：全量质检替代人工抽检，结构化输出接 BI
- **流式输出**：`Flux<String>` + SSE，提升用户体验
- **结构化输出**：BeanOutputConverter + JSON Schema 约束模型
