# 大厂面试备战计划（字节/蚂蚁/阿里）

> 目标：3 个月系统备战，算法 + 八股文 + 系统设计 + 项目全覆盖

---

## 一、各大厂面试风格

### 字节跳动
- 算法最难，4-5 轮全考，Medium/Hard 为主，甚至有 Hard+
- 八股文考原理，不只背答案，会追问"为什么这么设计"
- 节奏快，一面就开始做题

### 蚂蚁集团
- 金融背景，分布式/高可用/数据一致性是重点
- 系统设计比重大（设计支付系统、对账系统、限流组件）
- 算法也考但难度略低于字节

### 阿里（含淘天/云/菜鸟）
- 八股文 + 项目深挖结合
- 喜欢问"你们遇到什么问题、怎么解决的"
- P6 及以上对系统设计要求高

### 通用规律
- 至少 3 轮技术面 + 1 轮 HR
- 算法是硬门槛，一面挂了后面全没了
- 项目必须能被深挖：QPS、瓶颈、优化方案

## 二、3 个月备战计划

### 第 1 个月：算法攻坚（最重要）

| 周次 | 专题 | 目标题数 |
|---|---|---|
| 第 1 周 | 数组、链表、哈希表 | 30 题 Easy |
| 第 2 周 | 双指针、滑动窗口、二分 | 30 题 Easy/Medium |
| 第 3 周 | 树（DFS/BFS）、栈、队列 | 30 题 Medium |
| 第 4 周 | 回溯、排列组合 | 20 题 Medium |

**每日节奏**：1-2 题，先独立思考 20 分钟，超时看题解，第二天复刷

### 第 2 个月：算法进阶 + 八股文并行

**算法（每天 1 题）**

| 周次 | 专题 |
|---|---|
| 第 1 周 | 动态规划基础（爬楼梯/背包/最长子序列） |
| 第 2 周 | 动态规划进阶（区间DP/状态压缩） |
| 第 3 周 | 图论（拓扑排序/最短路径/并查集） |
| 第 4 周 | LeetCode Hot 100 查漏补缺 |

**八股文（每天 1 个专题）**

| 专题 | 核心考点 |
|---|---|
| JVM | 内存分区、GC 算法（G1/ZGC）、类加载、OOM 排查 |
| 并发 | synchronized/ReentrantLock、线程池 7 个参数、volatile、AQS 原理 |
| 集合 | HashMap 扩容（负载因子/红黑树转换）、ConcurrentHashMap 原理 |
| Spring | Bean 生命周期、AOP 动态代理（JDK vs CGLIB）、事务传播 7 种 |
| MySQL | B+树索引、事务隔离级别、MVCC、explain、慢查询优化 |
| Redis | 5 种数据结构、持久化（RDB/AOF）、缓存三问、分布式锁 |

### 第 3 个月：系统设计 + 模拟面试

**系统设计必练题目**

| 题目 | 考察重点 |
|---|---|
| 设计短链接系统 | 哈希冲突、高并发读、存储选型 |
| 设计限流组件 | 令牌桶/漏桶/滑动窗口算法 |
| 设计消息队列 | 可靠投递、顺序消费、重复消费 |
| 设计秒杀系统 | 库存扣减、防超卖、流量削峰 |
| 设计分布式 ID | Snowflake、号段模式 |

**模拟面试节奏**
- 每天 1-2 道算法限时 30 分钟
- 每周 1 次完整模拟面试（找朋友或用 AI 模拟）
- 复盘：哪道题思路卡了、哪个八股答得不好

## 四、项目包装（你的差异化优势）

大多数候选人没有 AI 工程项目经验，这是你的加分项，面试时主动引导：

**能说的技术亮点**
- RAG 全流程落地：分片策略、向量检索调优、多租户知识隔离
- Agent 实现：ReAct 模式、Function Calling 循环、Chain-of-Thought Prompt
- LLM-as-Judge：全量通话质检，结构化输出直接接 BI，conversionProbability 形成数据飞轮
- 流式输出：Flux<String> + SSE，首 token 延迟优化
- 结构化输出：BeanOutputConverter 约束 LLM 输出格式

**准备被追问**
- "你们 RAG 准确率怎么评估？" → 用 /rag/search 接口单独测检索质量，分离检索问题和生成问题
- "LLM 调用失败怎么处理？" → 重试机制、降级策略、超时设置
- "向量库选型为什么用 SimpleVectorStore？" → 学习项目，生产会换 Milvus/Weaviate/PGVector

## 五、八股文高频必背清单

### JVM（必考）
- [ ] JVM 内存分区（堆/栈/方法区/程序计数器）
- [ ] 四种引用类型（强/软/弱/虚）
- [ ] GC Roots 包含哪些
- [ ] G1 和 CMS 的区别
- [ ] 类加载双亲委派机制及破坏场景

### 并发（必考）
- [ ] synchronized 锁升级过程（偏向锁→轻量级锁→重量级锁）
- [ ] ReentrantLock 和 synchronized 的区别
- [ ] 线程池核心参数 + 拒绝策略
- [ ] ThreadLocal 内存泄漏原因
- [ ] happens-before 规则

### MySQL（必考）
- [ ] B 树和 B+ 树的区别，为什么选 B+
- [ ] 聚簇索引 vs 非聚簇索引
- [ ] 四种事务隔离级别 + 对应解决的问题
- [ ] MVCC 原理（ReadView + undo log）
- [ ] 为什么推荐用自增主键

### Redis（必考）
- [ ] Redis 为什么快（单线程 + IO 多路复用）
- [ ] 缓存穿透/击穿/雪崩的区别和解决方案
- [ ] 分布式锁 SETNX 的问题 + Redisson 怎么解决
- [ ] Redis 持久化 RDB vs AOF
