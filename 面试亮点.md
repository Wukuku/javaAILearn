# javaAILearn 项目面试亮点

## 一、技术栈定位（一句话介绍）

> 基于 Spring Boot 3 + Spring AI 1.0 的大模型 Agent 项目，涵盖从基础聊天到生产级 AI Agent 的完整链路。

---

## 二、六大模块递进式学习路径

| 模块 | 核心 API | 面试要点 |
|---|---|---|
| **chat** | `ChatClient` | 同步/流式双模式，流式用 `Flux<String>` + SSE |
| **memory** | `MessageChatMemoryAdvisor` | `conversationId` 多用户隔离；可无缝替换 Redis/JDBC |
| **structured** | `BeanOutputConverter` | `@JsonPropertyDescription` 自动生成 JSON Schema 约束模型输出 |
| **tools** | Function Calling | Spring AI 自动管理工具调用循环（ReAct 机制） |
| **rag** | `QuestionAnswerAdvisor` | TokenTextSplitter 分片、topK/相似度阈值调优、元数据过滤 |
| **agent** | 多工具协作 + CoT | Thought → Action → Observation 循环，强制 Chain-of-Thought |

## 三、最有深度：telesales 电销 AI Agent（综合所有技术）

### 1. LLM 替代 BERT 做意图分析
- BERT 痛点：单标签、新意图需重训练
- 升级：一次调用同时输出**主意图 + 复合意图 + 情感 + 紧迫度 + 推荐策略**，全部结构化 JSON

### 2. LLM-as-Judge 全量质检
- 人工抽检 < 5% → LLM 100% 覆盖
- `QualityReport` 直接写 DB 接 BI；`conversionProbability` 字段可作为训练数据**形成数据飞轮**

### 3. RAG 话术知识库
- 话术按意图 + 客户分层向量化，实时语义检索推荐，替代人工查手册

### 4. 全链路流式 Agent（`/simulation/stream`）
- 单请求串联：意图分析 → RAG 检索话术 → CRM Tool 查客户 → 流式生成个性化回复

## 四、工程细节（体现生产意识）

- **自定义 `LoggingAdvisor`**：切面式打印每次 AI 请求/响应，不侵入业务代码
- **`CallSessionManager`**：`ConcurrentHashMap` 管理会话，`AtomicInteger` 线程安全计轮次
- **`GlobalExceptionHandler`**：统一异常处理，标准 `ApiResponse` 封装
- **`docker-compose.yml`**：考虑了容器化部署

## 五、高频面试问题 & 回答方向

**Q: RAG 怎么调优？**
> 三个核心参数：`topK`（召回数量）、`similarityThreshold`（精度门槛）、Chunk 大小。项目里专门暴露了 `/rag/search` 接口，可以绕过 LLM 直接测检索质量。

**Q: 结构化输出原理？**
> `BeanOutputConverter` 把 Java Record + `@JsonPropertyDescription` 注解转成 JSON Schema，注入 Prompt 末尾，模型被约束必须返回合法 JSON，Spring AI 自动反序列化。

**Q: Agent 和普通对话有什么区别？**
> 核心是工具调用循环（ReAct）：模型返回 tool_calls → Spring AI 执行 Java 方法 → 结果作为 ToolMessage 回传 → 模型继续推理，可循环多次，直到模型主动结束。

**Q: 多轮对话如何做隔离？**
> `conversationId` 作为 key，每个会话独立存储历史。当前用 `InMemoryChatMemory`，生产换 `JdbcChatMemory` 只改一行 Bean，接口不变。
